# Code of Conduct

PaperTools contributors are expected to keep project discussions professional,
specific, and focused on improving the software.

## Expected Behavior

- Be respectful and direct.
- Assume technical disagreements are about the work, not the person.
- Give actionable feedback with enough context to reproduce or evaluate it.
- Protect private data, API keys, unpublished papers, and production logs.

## Unacceptable Behavior

- Harassment, threats, or discriminatory language.
- Publishing another person's private information.
- Posting secrets, credentials, or private production data.
- Repeatedly derailing technical discussions after maintainers ask you to stop.

## Enforcement

Maintainers may edit or remove comments, close issues, block accounts, or take
other reasonable action to protect the project and its users.
